Mission : LOG - Deploy (on a position)

Context

Set-up adopted by a unit for immediate ore later action.
In SWORD, to know how to order subordinate units to deploy onto a given deployment position. Once the subordinate agents get deployed, they activate their logistic chains.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Deployment position

Set of positions to use to self deploy (default value is 'EMPTY').

Behavior
  • The commander agent issues the "Deploy (on a position)" mission to the subordinate platoons, with the given position as parameter. Each platoon is given a position (the position is randomly chosen by the commander agent, in a circle around the given location).
  • If the given Deployment position is a ploygon, the commander agent splits the area into sub-areas and issues each of them to subordinate platoons that perform the "Deploy (on a position)" mission.
  • Other units are ordered to follow.

Specific cases

Comments

End of mission

The mission ends if another mission (or "the cancel maneuver" fragmantary order) is issued by user.
