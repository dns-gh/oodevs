Mission : COMMON - Move using an underground network

Context

To move to an objective using an underground network such as sewer, metro...
In Sword moving through a given underground network is simulated with a delay (the movement is not a visible behavior). This action is specific: to simulate the movement, the agent goes from one underground network gateaway (a sword object) to another one using a delay. Once the delay is over, the agent teleports itself on the exit gateway.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Underground entrance

Entrance to get in the underground network.

Underground exit

Exit to get out from the underground.

Objective

Final objective to reach.

Optional

Phase lines

Behavior
  • The commander agent issues the " Move using an underground network" mission to all subordinate units.
  • Upon delay termination, the subordinates are automatically teleported onto the Underground exit object, and then moves (normal movement) to the final objective.
  • No coordination is used in this mission. The subordinate units reach the objective at their own pace.

Specific cases

If at least one subordinate cannot do the "Move using an underground network" mission, then a report will be displayed ("All subordinates must be able to do the main tasks for this mission"), and the mission will end immediately.

Comments

End of mission

The mission ends when all units that perform the mission has reached the objective.
