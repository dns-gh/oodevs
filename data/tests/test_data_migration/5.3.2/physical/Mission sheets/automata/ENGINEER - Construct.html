Mission : ENGINEER - Construct

Context

To construct something.
In SWORD, to know how to create and build a set of planned objects that may be acted upon later on, or that may have an effect on the world. Thus, it is usually a mission preliminary to others.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Works

Set of objects to build.

Meeting position

Position to reach by the command post and units that don't participate to the bridging operation.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Wait for reinforcement

Define if the subordinate units need to wait for reinforcement when a work cannot be built (default value is FALSE).

Behavior
  • The commander agent issues the "Construct" mission to the engineer platoons, with a set of works as parameter. Works are fairly shared between engineer platoons.
  • Other units are ordered to reach the meeting position (if the number of engineer platoons is greater than the number of obstacles, non used platoons are ordered to reach the meeting position).
  • When all works have been completed, the engineer units move to the meeting position and the mission ends.

Specific cases
  • A work may not be built if one of the subordinate units doesn't have the physical capability or it hasn't enough resources to do it. In that case, two possibilities:
    • the mission parameter "Wait for reinforcement" is set to TRUE : a subordinate unit receive the "Contruct" mission with the object as parameter and wait to be reinforced (the company should be reinforced with adapted device),
    • the mission parameter "Wait for reinforcement" is set to FALSE (default value): the object is ignored.

Comments

End of mission

The mission terminates when all subordinates has finished their mission.
