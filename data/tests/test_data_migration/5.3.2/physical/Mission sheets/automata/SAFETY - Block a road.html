Mission : SAFETY - Block a road

Context

To construct roadblocks in order to control and filter movements of any kind of entity.
In SWORD, creating roadblocks allows the agent to prevent other agents through the road.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

RoadBlocks location

The roadblocks sites to build.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Remove installation (default value is TRUE)

Remove roadblock site at the end of the mission.

Behavior
  • The commander agent issues the "block a road" order to each capable agent, with one roadblock as parameter.
      • If there are strictly more roadblocks to perform than capable units, more objects will not be built.
      • If there are less roadblocks to perform than capable units, more capable units will be become support units.
  • Support units which have capability receive the "Follow and reinforce" mission with the agents wich have ordered "block a road" as parameter.
  • The other units receive the "Move to" mission with the planned objects as parameter.
  • When a construction site is completed, the agents stay at the construction site location.
  • At the end of the mission, the construction site is removed or not depend of parameter value "Remove installation"

Specific cases
  • The roadblocks may not be created if subordinate units do not have the capacity to build it regarding their physical capability and dotations. In that specific case, the report "work impossible, no capacity" is displayed by subordinate units.

Comments

End of mission

This mission continues until another one is issued.
