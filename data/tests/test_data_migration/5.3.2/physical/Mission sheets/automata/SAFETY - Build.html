Mission : SAFETY - Build

Context

To construct something.
In SWORD, to know how to create and build a set of planned objects that may be acted upon later on, or that may have an effect on the world. Thus, it is usually a mission preliminary to others.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Works

Set of objects to build.

Rally point

Position to reach after performing the mission.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Instantaneously

Defines if the subordinates will perform the works instantaneously (magically).

Wait for reinforcement

Defines if the subordinate units need to wait for reinforcement when a work cannot be built (default value is FALSE).

Behavior
  • The commander agent issues a "Perform Works" order to each capable agent. There are two cases :
    • If there are strictly more works to perform than capable units, all the units will be provided one obstacle. As soon as a unit has finished to build an obstacle, another obstacle will be assigned to him, unless there is no more obstacle left to build, in which case he will receive a *Move to* order to the rally point, or will help with other works if he can.

Assigning the first obstacles.

One subordinate will build the last obstacle, and the other subordinates will reinforce him.
    • Otherwise, the obstacles will all be provided to the units at the beginning of the mission. They will be fairly distributed among them. Multiple units can work on the same obstacle to work faster. Once a unit has finished its assigned work, he may help with other works if he can.

One subordinate will go build the first checkpoint, and the other two subordinates will build the second checkpoint.
  • When all works have been completed, the engineer units move to the meeting position and the mission ends
  • The other units are given a "Move To" order to the rally point.

Specific cases
  • A work may not be built if one of the subordinate units doesn't have the physical capability or it hasn't enough resources to do it. In that case, two possibilities:
    • the mission parameter "Wait for reinforcement" is set to TRUE : a subordinate unit receive the "Contruct" mission with the object as parameter and wait to be reinforced (the company should be reinforced with adapted device),
    • the mission parameter "Wait for reinforcement" is set to FALSE (default value): the object is ignored.

Comments

End of mission

The mission terminates when all subordinates has finished their mission.
