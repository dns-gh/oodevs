Mission : SAFETY - Create and operate a checkpoint

Context

To operate a checkpoint in order to control and filter movements of any kind of entity.
In SWORD, creating and operating a Checkpoint allows the agent to filter crowds. Filtering crowds allows the agent to detect hidden units (terrorist or other characters).

Parameters

Mandatory

Danger direction

Expected enemy direction.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Checkpoint location

The checkpoint to build and operate.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Remove installation (default value is TRUE)

Remove decontamination site at the end of the mission.

Intensity (%, default is 100%)

The strength that is used to filter crowds. A value of '0' means "no filtering at all". A value of '100' means "totally blocked" (the crowds that interact with the checkpoint cannot go through it).

Behavior
  • The commander agent issues the "Create and filter" mission to one of platoon, with the planned site as parameter.
  • Support units receive the "Follow and reinforce" mission with the platoon which have the mission "Create and filter" as parameter.
  • The other units receive the "Move to" mission with the planned object as parameter.
  • When the site construction is completed, the agents which have the capabilities to operate a check point move to the center of the objet and start to operate it.
  • Other units move to the center of the object and operate it if they have the capabilities.

Specific cases
  • The checkpoint may not be created if subordinate units do not have the capacity to build it regarding their physical capability and dotations. In that specific case, the report "work impossible, no capacity" is displayed by subordinate units.
  • The checkpoint not be operated if it cannot be operated (see object capacity "workable" in authoring tool), In that specific case, the report "element cannot be operated" is displayed by subordinate units.
  • Note that no specific capabilities are requeried for the subordinates units to operate a checkpoint.

Comments

End of mission

This mission continues until another one is issued.
