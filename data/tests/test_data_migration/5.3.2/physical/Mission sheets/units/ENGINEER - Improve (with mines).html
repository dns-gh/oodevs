Mission : ENGINEER - Improve (with mines)

Context

To improve natural or man made obstacles.
In Sword, to improve a set of sword objects with mines (to add mines tp the object). An improved object (an engineer obstacle for instance) is harder to remove because agents of the simulation need to remove mines before deconstructing the object. Remove mines improvement takes delays.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Objectives

Objects to improve with mines.

Meeting position

Position to reach upon works termination.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Behavior

The agent moves toward each obstacle to improve. It acts on each obstacle by following the order of the elements in the 'Objectives' list. The position of each obstacle to improve is automatically determined by the agent.
  • Upon reaching an object, the agent starts to add mines (the report "Valorization started for object: [object name]" should be displayed). Upon work termination (the report "Improved completed for object: [object name]" should be displayed) and the agent goes to next object.
  • An object may not be improved if the agent doesn't have the physical capability or it hasn't enough resources. In this case, the report 'Impossible work' is displayed.
  • In this case the agent waits to be reinforced.
  • If needed, the agent dismounts to improve the obstacle.
  • When all obstacles have been improved, the agent moves to the meeting position.

Specific cases

Comments

End of mission

The meeting position is reached.
