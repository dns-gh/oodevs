Mission : SAFETY - Damage urban blocks within area

Context

To damage installations.
In SWORD, to damage a set of urban blocks within a zone, by decreasing their structural state. Damages here are intended to mean minor destruction (deconstruction) of urban blocks. This mission may be used to model terrorist attacks, violent demonstrations or riots. A damaged urban block has the effect of making resource nodes less efficient (production decreases, for instance).

Parameters

Mandatory

Danger direction

Expected enemy direction.

Area

The area inside which urban blocks must be damaged.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Behavior

The agent moves toward each urban block which is totally inside the specified area (the order of traversal is randomly defined). If a visited urban block is reachable (the agent can enter it), the agent moves to the center, otherwise it moves to a close position (the position closest to the center).
Upon reaching an urban block, the agent starts to deconstruct it during a limited time. The deconstruction time depends on the time required for agent installation (see the authoring tool in "Stances" section, "Time to activate" ). The report "deployment in progress" and "work just started" should be displayed, meaning that the agent is damaging the target. Once the report "deployment complete" is displayed, the time has come for the agent to visit the next urban block.

Specific cases

An urban block may not be damaged if:
  • the agent doesn't have the physical capacity to deconstruct urban blocks,
  • the agent's stances defined in the authoring tool doesn't permit enough time for the agent to damage its target.
In this case, the agent decides to act on the next one.

Comments

End of mission

When there are no more urban blocks to visit, the agent stays next to the last visited block and the mission ends.
If there are no urban blocks within the area, the mission ends and the report "no urban blocks within area" is displayed.
