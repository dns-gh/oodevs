Mission : SAFETY - Perform construction works

Context

To construct something.
In SWORD, to know how to create and build a set of planned objects that may be acted upon later on, or that may have an effect on the world. Thus, it is usually a mission preliminary to others.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Works

the works to perform,

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Wait for reinforcement (default value is FALSE)

defines if the agent needs to wait for potential reinforcement in case it cannot do a work.

Instantaneously (default value is FALSE)

define if agent will build issued elements (objects or urban blocks) instantaneously (magically).

Behavior

The agent moves toward each planned work to create (the order of traversal is defined by user). The positions reached to create each object are automatically determinated by the agent.
Upon reaching a construction position, the agent starts to create and construct the corresponding object (the "work just started" report should be displayed). Upon construction termination (the report "work completed for object [object name]" should be displayed), the agent moves to the next work to do (next construction position for the next objet to create).

Specific cases

If the parameter "Instantaneously" is set to TRUE, the agents creates and build each issued work magically. In that specific case, the object construction do not depends on the agent's physical capabilities or the object definition.
A work may not be created. The possible causes are:
  • the agent doesn't have the physical capability or if it hasn't enough resources to do it.
  • the planned work cannot be created because it is not designed for (see object capacity "construction" in authoring tool),
If that specific case, the agent behaves in two ways depending on the 'waitForReinforcement" parameter:
  • if it is set to false, the report 'Impossible work' is displayed, and the agent moves to the next work to create;
  • if it is set to 'true', agent will stay nearby the planned object and ask for reinforcement. It won't move to the next work until it is created.

Comments

If needed, the agent dismounts to build an object.

End of mission

The mission is done when all works have been visited (even if some of the objects couldn't be built).
