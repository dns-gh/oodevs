Mission : SAFETY - Operate a checkpoint (filter crowds)

Context

In SWORD, checkpoints are object that can filter crowds. Using such an object allows the agent to filter crowds in order to detect hidden units (terrorist or other characters) and prevents them from going into restricted areas.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Checkpoint

the object to be used to filter crowds,

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Intensity (%, 100 by default)

the intensity to apply to filter crowds, 100% by default.

Behavior

The agent moves to the center of the object. Then, filtration starts with the given intensity (the trace "check point established" should be displayed). An intensity value of 100% means that agent will filter with the max intensity regarding to properties defined on the "check point " object (see the "population/density" capacity in authoring tool). For instance, a "Density" value of "0 " means that a filtration of 100% will block all crowds trying to go through the checkpoint. An intensity value of 50% means that the density of crossing crowds will be devided by two.
The agent remains there until another mission is issued.

Specific cases

The object may not filter crowds if its properties are not well defined (see the object capacity "population" in authoring tool). In that specific case, no specific report is displayed.

Comments

End of mission

When there is no more waypoints to visit, the agent stays next to the last one and the mission ends.
