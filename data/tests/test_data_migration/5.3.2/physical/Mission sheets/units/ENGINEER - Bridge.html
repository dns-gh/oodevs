Mission : ENGINEER - Bridge

Context

The purpose of any river crossing is to project combat power across a water obstacle to accomplish a mission. The challenge is to minimize the river's impact on the commander's ability to maneuver.
In Sword, to use a crossing site object to construct a "bridge" object in order to allow friendly units to cross a water obstacle.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Crossing site

Crossing site (object) to bridge.

Bridge system

Define if the bridging system must be discontinuous or continuous (float bridging or fixed bridging).

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Behavior
  • To bridge a water obstacle the crossing site object must have been reconnoitered. If this is not the case the report "crossing site needs to be recon first" is displayed and the mission terminates.
  • If the crossing has been reconnoitered, the agent moves to it and builds a "bridge" object on it (the constructed bridge has the same shape as the crossing site).
  • Note that the agent may receive reinforcements in order to build the bridge. The bridge becomes operational once the report "bridge is operational" displays.

Specific cases

Comments

End of mission

The mission ends if another mission (or "the cancel maneuver" fragmentary order) is issued by user.
