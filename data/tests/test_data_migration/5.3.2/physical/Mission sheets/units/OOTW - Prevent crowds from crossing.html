Mission : OOTW - Prevent crowds from crossing

Context

To operate a checkpoint in order to deny the crossing of crowds.
In SWORD, to operate a checkpoint (a Sword object can have the ability to filter crowds) with a high blocking strength (all encountered crowds are stopped).

Parameters

Mandatory

Danger direction

Expected enemy direction.

Checkpoint position

The position at which the agent creates the checkpoint.

SafetyPosition

The position to reach when the agent cannot stand in front of uncontrolled crowds.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Behavior
  • The agent moves to the work location and creates (magically) a checkpoint (the report "control point established" should be displayed)
  • The agent then moves to the center of the checkpoint in order to operate it (the report "element [object name] is being operated should be displayed).

Specific cases
  • If the work location is not reachable, the agent moves as close as possible, otherwise it moves to a close position (the closest position from the center of the future object) and the agent builds the object. In this case, it is possible that the object will not be built at the exact location.
  • If encountered crowds cannot be controlled (see specific reports) the checkpoint may be destroyed and the agent may protect itself by moving to the given safety position:
1) The agent tries to deny the crowd from crossing the crossroad:

2) The agent cannot stand in front of the crowd:

Comments

End of mission

This mission can end in two ways:
  • encountered crowds can be kept under control. In this case the agent remains at the checkpoint until mission termination.
  • an encountered crowd could not be kept under control. In this case the agent reaches the safety position and the mission terminates.
