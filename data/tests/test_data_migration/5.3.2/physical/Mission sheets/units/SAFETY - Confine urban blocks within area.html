Mission : SAFETY - Confine urban blocks within area

Context

In the event of an exterior threat (a fire, for instance), populations may be prevented from leaving the buildings they occupy.
In SWORD, confining a urban block prevents the population from leaving or entering it. A urban block cannot be confined if it cannot be inhabited. An urban block has a "confined" status ("Yes" or "No").

Parameters

Mandatory

Danger direction

Expected enemy direction.

Area

The area inside which urban blocks must be confined.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Behavior

The agent moves toward each urban block which is totally inside the specified area (the order of traversal is randomly defined). The agent stops at the border of the urban block.
Upon reaching a urban block, the agent confines it before moving to the next unvisited urban block.

Specific cases

An urban block may not be confined if:
  • the urban block cannot be inhabited (the report "urban block cannot be inhabited" is displayed) ,
  • the urban block is already confined (the trace "the urban block is already confined" is displayed).
In this case, the agent decides to act on the next one.

Comments

End of mission

When there are no more urban blocks to visit, the agent stays next to the last block visited and the mission ends.
If there are no urban blocks within the area, the mission ends and the report "no urban blocks within area" is displayed.
