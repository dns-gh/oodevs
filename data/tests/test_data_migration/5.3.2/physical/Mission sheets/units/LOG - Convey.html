Mission : LOG - Convey

Context

In Sword, for a logistics agent (see type of agent in the authoring tool), to automatically supply recipients that have asked for resources. It is automatically issued by the Sword simulation when convoys are created if units ask to be re-supplied.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Behavior
  • The visible behavior that is performed by the agent is described as follows:
    • the agent waits for its vehicle to be set up (another agent provides vehicles to create the convoy). The report "convoy is being prepared" is displayed.
    • Then, the agent moves to the supplier, and loads supplies (the reports "moving to supplier" and then "convoy: loading" are displayed).
    • Once it has finished, it moves to the recipient and unloads supplies (the reports "moving toward unloading point" and then "convoy: unloading" are displayed).
    • It then moves to the vehicle provider (the agent that set up the convoy) and the mission ends and terminates.

Specific cases
  • In case of pushed flux, it possible for the convey to move and suplliy several recipients. In that case, the agent moves to the first one, unloads supplies, then moves to the second one and so on...

Comments
  • When the mission terminates, the created convoy "disappears" from the orbat (the vehicles return inside provider, and the convoy no longer exists).
  • The user can terminate the mission by issuing another mission or a fragmentary order. The user can then reissue the mission (by left-clicking on the unit) and the agent will be able to continue the resupply action.
  • For more information about the stages of the supply actions, you can open the the supply system window (see screenshot). From here you will be able to follow the processing consigns (by double-clicking on each consign).

End of mission

When the convoy has returned back the vehicle provider (the unit which has created the convoy).
