Mission : ENGINEER - Construct

Context

To construct something.
In SWORD, to create and build a set of planned objects that may be acted on later on, or that may have an effect on the world. Thus, it is usually a mission preliminary to others.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Works

Works to perform.

Meeting position

Position to reach upon action termination.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Wait for reinforcement

Define if agent needs to wait for reinforcement when it is not able to perform work (false by default).

Behavior
  • The agent moves toward each work to construct. The order of visit is from the nearest to the furthest. The positions reached to construct each object are automatically determined by the agent.
  • Upon reaching a construction position, the agent starts to construct the object (the "work just started" report should be displayed). Upon construction termination (the report "work completed for object[object name]" should be displayed), the agent moves to the next work to do (next construction position for the next objet to construct).
  • In obstacle contruction, if the parameter "Improve with mines" is set to TRUE, the agent improves (or completes the improvement of) each obstacle with mines.
  • If needed, the agent dismounts to construct an object.

Specific cases

A work may not be constructed, if the agent doesn't have the physical capability or hasn't enough resources. In this case, the agent behaves in two ways depending on the "waitForReinforcement" parameter:
  • if it is set to FALSE, (default value) the report 'Impossible work' is displayed, and the agent moves to the next work to construct;
  • if it is set to TRUE, the agent will stay nearby the object and ask for reinforcements. It won't move to the next work until it is totally constructed.
  • At any times, the user can order the agent to visit the next obstacle by issuing the "Go to next objective" fragmentary order . If this order is issued when the agent is working on the last work, the agent moves to the meeting position.

Comments

End of mission

The mission is complete when all works have been visited; the agent then moves to the meeting position and the mission terminates.
