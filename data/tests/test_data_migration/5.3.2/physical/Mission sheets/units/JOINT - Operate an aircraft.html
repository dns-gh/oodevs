Mission : JOINT - Operate an aircraft

Context

In this Sword mission, to move to a given position and operate an aircraft. Thus, it is usually a mission preliminary to the UAV "JOINT - Bomb (enemies within an area)" mission.

Parameters

Mandatory

Danger direction

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Position

Position at which the agent operates an aircraft (empty by default).

Behavior
  • The agent moves to the issued position.
  • If no position issued, the agent stays at its own position.
  • Upon reaching the position, the agent self deploys in order to operate one of its aircraft ( the reports "starting deploy" and then "section deployed" should be displayed; for the drone "aircraft is being operated" and" aircraft has been operated" ). The time required for deployment depends on the data defined in the authoring tool (see "Units" tab, "Deployment" section).
  • Note that the agent chooses an aircraft among those that are available to be operated. The corresponding aircrafts are the those that have not been already operated, or the those that are not already performing a mission. If the agent cannot operate an aircraft, the report "no aircraft available" is displayed. In that case the mission terminates.
  • Upon deployment termination, the agent has operated an aircraft that will be ready to receive a mission. To verify that an aircraft agent can perform a mission, the aircraft agent should display "aircraft ready".

Specific cases

Comments

End of mission

Once an aircraft has been operated, the mission ends. The agent can continue to operate other aircraft using this mission.
