Mission : SAFETY - Light up fires

Context

Arsonists or pyromaniacs may start fires in order to attack installation or induce terror on populations.
In SWORD, to create a set of planned fires at different locations. Thus, it is usually a mission performed by terrorists, arsonists or pyromaniacs.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Fires locations

the planned fire to create,

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Instantaneously (default value is FALSE)

defines if the agent can create fires instantaneously (magically) without any physical capabilities consideration

Rally point

Behavior

The agent moves toward each fire location. The positions reached to create each fire are automatically determinated by the agent. The order of the elements into the 'Fires locations' list defines the order of visit.
Upon reaching a position, the agent starts to create the corresponding object ('arson' or 'fire' object). The "work just started" report should be displayed. Upon creation termination (the report "work completed for object [object name]" should be displayed), the agent moves to the next fire to start.

Specific cases

A fire may not be created. The possible causes are:
  • the agent doesn't have the physical capability or if it hasn't enough resources to do it( if the 'Instantaneously' paramteter is set to FALSE).
  • the planned fire cannot be created because it is not designed for (see object capacities "construction" and "burn" in authoring tool),
If that specific case, the report 'Impossible work' is displayed, and the agent moves to the next fire to create.

Comments

If needed, the agent dismounts to build an object.

End of mission

The mission is done when all fire have been visited (even if some of the objects couldn't be built).
