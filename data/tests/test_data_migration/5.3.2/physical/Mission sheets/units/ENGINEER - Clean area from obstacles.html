Mission : ENGINEER - Clean area from obstacles

Context

To reduce or negate the effects of existing or reinforcing obstacles. The objective is to maintain freedom of movement for maneuver units, weapons systems and critical supplies.
In Sword, to remove a set of given obstacles (such as mines, barricades etc.) in a given area.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Area

Area to clean.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Wait for reinforcement

Defines if agent should wait for reinforcement in case it cannot perform the cleairing action on an obstacle.

Behavior
  • The agent moves to the area and reconnoiters it. This stage should allow the agent to discover objects within the area.
  • Upon reconnaissance termination, the agent moves toward each obstacle to remove (all objects may not be considered 'obstacles'). Note that order of visit is randomly determined. The position from which to remove each obstacle is automatically determined by the agent. If an obstacle is improved with mines, the agent needs to remove mines before starting to deconstruct it (and therefore causes more delays).
  • Upon reaching an obstacle, the agent starts to remove it (the report "work just started" should be displayed). Upon work termination (the report "work completed for object [object name]" is displayed) and the agent goes to next obstacle.
  • An obstacle may not be removed if the agent doesn't have the physical capability or it hasn't enough resources. In this specific case, the report 'Impossible work' is displayed, and the agent reaches the next obstacle to remove.
  • If needed, the agent dismounts to remove the obstacle.
  • When all obstacles have been removed, the agent moves to the center of the area and the missions ends.

Specific cases
  • If the "wait for reinforcement" parameter is set to "true" (default vlaue is 'FALSE') the agent waits for reinforcement before visiting the next obstacle to remove.
  • At any times, the user can order the agent to visit the next obstacle by issuing the "Go to next objective" fragmentary order . If this order is issued when the agent is working on the last obstacle, the agent moves to the center of the area and the mission terminates.

Comments

'Obstacles' are "buildable", "removable" objects, such as defined in the authoring tool ('objects' tab, 'capacities' session).

End of mission

When all known obstacles has been visited (and may be removed or not) into the provided area.
