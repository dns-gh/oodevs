Mission : ENGINEER - Open a route

Context

To reduce or negate the effects of existing or reinforcing obstacles. The objective are to maintain freedom of movement for maneuver units, weapons systems and critical supplies.
In Sword, to move along a computed path and bypass obstacles that collide with the path. Here, 'obstacles' are sword 'objects' that have the capacity "avoidable" and/or "constuctor".
Bypassing an obstacle means to make it trafficable. Agents of the simulation becomes able to move through a bypassed obstacle with a limited speed of movement.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Waypoints

Waypoints defining the route to follow (the path will be computed taking into account these waypoints).

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent perform the mission.

Limit 2

Right or left boundaries inside which the agent perform the mission.

Behavior
  • The agent moves along the computed path. If an 'obstacle' obstructs the path, the agent tries to reaches it.
  • Upon reaching an obstacle, the agent tries to bypass it:
    • the obstacle can be bypassed: the agent starts to bypass it (the report "work just started" should be displayed),
    • the obstacle can be bypassed but the agent has no capacity to perform the action: the report "the agent cannot bypass the object" should displayed and the agent waits for reinforcement.
    • the obstacle cannot be bypassed: the report "the object cannot be bypassed" should display.
  • In the last case, the agent tries to remove the object: if an obstacle is improved with mines, the agent needs to remove mines before starting to deconstruct the obstacle, then the agent starts to remove it (the report "work just started" should be displayed). Upon work termination (the report "work completed for object [object name]" is displayed) the agent continues to move along the path.
  • If the obstacle cannot be removed either (if the agent doesn't have the physical capability or it hasn't enough resources), the report 'Impossible work' is displayed. In that case the agent waits for reinforcement.
  • Upon work termination (the report "work completed for object [object name]" is displayed) the agent continues to move along the path.
  • If needed, the agent dismounts to perform the works.

Specific cases
  • Note that the wait can be very long (it can depend on the type and the shape of the obstacle).
