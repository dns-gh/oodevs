Mission : COMMON - Move within an underground network

Context

To move inside an underground network such as a sewer, metro...
In Sword moving through a given underground network, the agent goes from one underground network gateway (a sword knowledge object) to another using a time period. Once the period is over, the agent teleports itself to the exit gateway. In this mission, the agent doesn't exit from the underground.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Gateway

Next network station to reach.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Behavior
  • While moving through the network, a report informing the user how much time the movement will take is displayed ("arrival in 20minute(s)" for instance). A specific drawing is also displayed on the agent's symbol (see screenshot below).
  • Upon delay termination, the agent is automatically teleported onto the Underground exit object, gets outside and then moves (normal movement) to the final objective.

Specific cases

The agent won't be able to move through the underground network if:
  • the next gateway to reach is destroyed (the knowledge object disappears)
  • the given gateway to reach does not belong to the same network as the one the agent is currently using ( the report "underground network issue is not operational (state inactivated) or the unit is already underground" is displayed).
