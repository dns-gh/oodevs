Mission : ENGINEER - Bypass obstacles

Context

To reduce or negate the effects of existing or reinforcing obstacles. The objective is to maintain freedom of movement for maneuver units, weapons systems and critical supplies.
In Sword, to bypass a set of sword objects. Bypassing an obstacle means to make it trafficable. Agents of the simulation becomes able to move through a bypassed obstacle with limited speed of movement.

Parameters

Mandatory

Danger direction

Expected enemy direction.

Obstacles

Objects to bypass.

Meeting position

Position to reach upon works termination.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Wait for reinforcement

Defines if agent should wait for reinforcement in case it cannot perform the bypassing action on an obstacle.

Behavior
  • The agent moves toward each obstacle to bypass. It acts on each obstacle by following the order of the elements in the 'Obstacles' list. The position to bypass each obstacle is automatically determinated by the agent.
  • Upon reaching an obstacle, the agent starts to remove mines if the obstacle is improved by them (the report "demining the object [object name]" should be displayed). Then, three possibilities:
    • the obstacle can be bypassed: the agent starts to bypass it (the report "work just started" should be displayed).
    • the obstacle can be bypassed but the agent has no capacity to perform the action: the report "the agent cannot bypass the object" should displayed.
    • the obstacle cannot be bypassed: the agent starts to remove it (the report "work just started" should be also displayed). If the obstacle cannot be removed either (if the agent doesn't have the physical capability or it hasn't enough resources to do it), the report 'Impossible work' is displayed. In this case the agent waits to be reinforced.
  • Upon work termination (the report "work completed for object [object name]" is displayed) the agent goes to next obstacle.
  • If needed, the agent dismounts to perform the work.
  • When all obstacles have been bypassed, the agent moves to the meeting position.

Specific cases
  • Note that the bypassing delay can be very long (it can depend on the type and shape of the obstacle).
  • If the "wait for reinforcement" parameter is set to "true" (default vlaue is 'FALSE') the agent waits for reinforcement before visiting the next obstacle to bypass.
  • At any times, the user can order the agent to visit the next obstacle by issuing the "Go to next objective" fragmentary order . If this order is issued when the agent is working on the last obstacle, the agent moves to the meeting position.

Comments

End of mission

The meeting position is reached.
