Mission : COMMON - Follow

Context

To follow a set of mobile agents (units or crowds).
In this mission, the agent will follow only operational agents (for units) or crowds (alive).

Parameters

Mandatory

Danger direction

Expected enemy direction.

Entities

Entities to follow.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Distance

Behavior
  • The agent chooses and moves to a position in order to stay as close as possible to the given set of agents. If the position is not reachable (an urban block inside which the agent cannot move, for instance), the agent moves as close as possible, otherwise it moves to the center of the position. Average speed depends on the physical capabilities, taking into account terrain properties.
  • The chosen position changes when one of these two conditions is met :
    • The agent has reached the position
    • The followed unit has covered a distance greater than the distance that the agent can cover in forty seconds at its maximum speed.

Specific cases
  • If no entities can be followed (none of them are operational or alive), the mission ends with a 'failed' status.
  • An entity may not be followed if it is not operational (for units) or alive (for crowds). In this case, the report "The followed unit is destroyed" is displayed. If none of the issued agents can be followed, the mission ends with a 'failed' status, displaying the report "No valid parameter".

Comments

End of mission

The mission ends if another mission (or "the cancel maneuver" fragmentary order) is issued by user.
