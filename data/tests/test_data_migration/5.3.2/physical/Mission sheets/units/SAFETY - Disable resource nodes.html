Mission : SAFETY - Disable resource nodes

Context

To stop supplying the population with essential resources in order to prevent a disaster (avoid pollution) or provoke a disaster (terrorist attack).
In SWORD, disabling an enabled node deactivates the transfer of resources between nodes. It can be used to simulate a terrorist attack ("destruction" of a node) or a preventive deactivation by authorities (water pollution for instance).

Parameters

Mandatory

Danger direction

Expected enemy direction.

Resource nodes

The set of resource nodes to disable.

Optional

Phase lines

Lines used by units to adapt their behavior when they reach or cross them.

Limit 1

Right or left boundaries inside which the agent performs the mission.

Limit 2

Right or left boundaries inside which the agent performs the mission.

Behavior

The agent moves toward each of the resource nodes (a resource node is located within its corresponding urban block). If the urban block containing the resource node to deactivate is reachable (the agent can enter it), the agent moves to the center, otherwise it moves to a close position (the position closest to the center). The order of the elements in the 'Resource nodes' list defines the order of visit.
Upon reaching a resource node (the urban block containing it), the agent deactivates it, displaying the report "Resource node disabled". The agent then decides to act on the next one.

Specific cases

Comments

End of mission

When there are no more resource nodes to disable, the agent stays next to the last visited node and the mission ends.
